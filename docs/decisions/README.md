# Decision Records

Use this directory for gateway-local architecture decisions.

Filename pattern:

- `YYYY-MM-DD-<topic>.md`

Expected statuses:

- `Proposed`: directional change under active design
- `Accepted`: approved and expected to guide implementation
- `Superseded`: replaced by a later decision record

Each record should stay short and decision-oriented:

- what is changing
- what is being rejected or reconsidered
- what is already locked
- what is still open
- what must be true before promotion to `Accepted`
