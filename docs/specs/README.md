# Specifications

The active spec set is intentionally short and structural.

- `01_cli_surface.md` defines the CLI surface and domain split.
- `02_container_assets.md` defines the appliance container asset taxonomy.
- `03_runtime_stack.md` defines runtime responsibilities.
- `04_tools_lifecycle.md` defines the tooling domain lifecycle.
- `05_deployment_pipeline.md` defines deploy and rollback expectations.
- `06_semantic_router.md` defines the Semantic Router and escalation ladder.
- `08_cli_refactor_control_plane.md` defines the proposed CLI and control-plane refactor before implementation.
- `09_runtime_monitor_api.md` defines the proposed runtime monitor and operator API contract.
- `10_cli_refactor_implementation_plan.md` defines the phased rollout plan for the CLI refactor.
