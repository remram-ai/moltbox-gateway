# Runbooks

Use this directory for operator-facing procedures where the human needs to do a bounded set of real steps.

Runbooks should be:

- explicit
- sequence-driven
- safe for a non-expert operator to follow
- tightly scoped, with a clear stop condition

For the current clean-box effort, the primary manual runbook is:

- `2026-04-04-zfs-rebuild-ssh-takeover-runbook.md`

That runbook exists to minimize human keyboard time and hand control back to the AI builder as fast as possible.
